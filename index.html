Contextualización de bases de datos

¿Qué es un motor de bases de datos?

Los motores de bases de datos son programas intermedios entre las bases de datos y la aplicación que usa la base de datos, cada motor se encarga de una tarea en especifica como crear la base de datos o administrar el acceso a esta. Los motores de bases de datos optimizan y agilizan el proceso de acceso, consulta y extracción de datos en un software, estos deben cumplir con las siguientes caracteristicas:

  • Abstracción de la información
  • Independencia
  • Redundancia mínima
  • Consistencia
  • Seguridad Integridad
  • Respaldo y recuperación
  • Control de la concurrencia
  • Tiempo corto de respuesta.

Los motores de bases de datos son sistemas internos que ayudan de manera óptima a la creación de las bases de datos, estos estan integrados en los gestores de bases de datos que son el software que nos facilita conectarnos con el motor de bases de datos. Entre los gestores de bases de datos más conocidos están MySQL Server, SQL Server, Oracle, Microsoft Access, entre otros.

¿Qué características tiene una base de datos relacional?

Las bases de datos relacionales son las más usadas actualmente en todo el mundo, también son llamadas bases de datos SQL, estas almacenan datos relacionados entre sí, como su mismo nombre lo dice, donde los datos son representados sin necesidad de jerarquizarlos.

Las bases de datos relacionales cuentan con las siguientes características:

  • Son u conjunto de tablas formadas por filas donde cada fila debe tener un identificador clave (ID)
  • Los ID facilitan el acceso a la información
  • Se evitan la duplicidad de registros
  • Si un registro es eliminado no deben verse afectados los restantes
  • No pueden existir dos tablas con el mismo nombre

Ventajas de usar bases de datos SQL

  • Permite el manejo de grandes cantidades de datos de manera segura.
  • Garantizan la prevención de duplicidad de registro.s
  • Se puede bloquear el acceso a ciertos datos mientras estos se actualizan.
  • Permite el consumo de datos de varios usuarios al mismo tiempo a traves de la concurrencia.

Desventajas de usar bases de datos SQL

  • Deficientes a la hora de manejar datos gráficos o multimedia.
  • No permiten crear tablas de forma jerárquica

¿Qué es una base de datos no relacional?

Las bases de datos no relacionales nacieron a causa de los problemas que generaban el uso de bases de datos relacionales al momento de gestionar grandes cantidades de datos, haciendo que se ralentizara su funcionamiento. Las bases de datos no relacionales o también llamadas NoSQL no cumplen con el esquema entidad-relación, tampoco hace uso de tablas, sino que usa formatos como clave-valor, mapeo de columnas o grafos.

Una característica particular de este tipo de bases de datos es que no usan lenguaje SQL para la consulta de datos, sin embargo, si se puede usar SQL como herramienta de apoyo. Entre los tipos de bases de datos no relacionales se encuentran las bases de datos en grafo, orientadas a objetos, documentales, y clave-valor.

Usar bases de datos no relacionales tiene ciertas ventajas:

  • La información se almacena en documentos y no en tablas
  • Se ejecutan en dispositivos con pocos recursos
  • Son útiles cuando no se tiene claro el tipo de datos a almacenar
  • Manejan gran cantidad de datos
  • No generan cuellos de botella en las consultas

¿Cuáles son las bases de datos no relacionales que se manejan en la actualidad?

Algunos ejemplos de gestores de bases de datos no relacionales son los siguientes:

  • Cassandra

    Es una base de datos creada por Apache y es de tipo clave-valor. Cuenta con su propio lenguaje para realizar consultas llamado Cassandra Query Language, tambien fue desarrollada en Java, lo que permite ejecutarse cualquier plataforma. Cassandra es usada por empresas grandes como Facebook y Twitter.

  • Redis

    Es una base de datos no relacional creada por Salvatore Sanfilippo y Pieter Noordhuis. Es usada por empresas como Flickr, Instagram y Github con el tipo clave-valor. Redis no permite realizar consultas, sólo se puede insertar y obtener datos. Redis es compatible con Linux, Unix y OS, sin embargo, aun no existe soporte oficial para Windows.

  • MongoDB

    Usada por empresas como FourSquare, SourceForge, CERN, es una base de datos no relacional y una de las favoritas de los desarrolladores, es una base del tipo orientada a documentos, donde cada entrada puede tener un esquema de datos diferente que no tenga que ver con el resto de los registros, a este proceso se le llama esquema libre. MongoDB esta desarrollada en C++ haciéndola una base de datos ágil.

  • CouchDB

    Es una base de datos creada por Apache y soportada en sistemas Linux, POSIX, OS, pero no en Windows. JavaScript es su principal lenguaje de interacción y usa Restfull HTTP API como interfaz. Permite realizar operaciones típicas de SQL llamadas JOIN y para el almacenamiento de datos utiliza el formato JSON.

¿Cuáles son los motores de bases de datos relacionales licenciados más utilizados en el desarrollo de aplicaciones?

Entre los motores y gestores de bases de datos relacionales más usados en el mundo se encuentra:

  • MySQL

    Uno de los más usados en páginas web actuales y en aplicaciones de software libre. Es de facil instalación, configuración y uso, con un gran rendimiento y la capacidad de tener soporte multiplataforma y SSL. Una de sus deficiencias es su baja escalabilidad, siendo poco eficiente trabajando con grandes cantidades de datos que superen un cierto tamaño.

  • MariaDB

    MariaDB es uno de los gestores de bases de datos más usados en la actualidad. Unas de sus principales características son el aumento de motores de almacenamiento, gran escalabilidad, cuenta con buena seguridad y rapidez en las transacciones. Algunas de sus pocas desventajas son algunas incompatibilidades en la migración de MariaDB a MySQL.

  • Es una biblioteca open source utilizada en gran cantidad de aplicaciones de la actualidad, al ser una biblioteca resulta ser mas liviana, también logra gran estabilidad con su automicidad, consistencia, aislamiento, portabilidad, rendimiento y durabilidad.

  • PostgreSQL

    Es un motor de bases de datos orientado a objetos estable, robusto y eficiente con licencia BSD. Cuenta con control de concurrencias multiversión (MVCC), es flexible en cuanto a lenguajes de programación y multiplataforma. Tambien cuenta con una herramienta muy útil e intuitiva para la administración de bases de datos.

  • Microsoft SQL Server

    Es un gestor de bases de datos capaz de poner a disposición datos a muchos usuarios al mismo tiempo. Es un sistema de Microsoft con gran escalabilidad, estabilidad y seguridad con la posibilidad de cancelar consultas y un potente entorno gráfico de administración, además, permite usarse en Linux o Docker sin importar que sea nativo para Windows.

  • Oracle

    Oracle es uno de los sistemas gestores de bases de datos más robustos y completos, siendo de los preferidos en el mundo laboral. Entre sus ventajas se encuentra el soporte de transacciones, estabilidad, escalabilidad y multiplataforma. Una de sus desventajas es que sus principales opciones de uso son pagas, sin embargo, cuenta con una versión gratuita con menos herramientas.

¿Cuáles son los motores de bases de datos relacionales Open Source?

Se define Open Sorce como el software de código abierto, es decir, software cuyo código sea público y este a la dispocisión de ser modificado, visto o distribuido por cualquier persona.

Los cambios que se realizen del código deben ser revisados antes de implementarlos en el sistema en general, estas revisiones se hacen de manera colaborativa por desarrolladores. En el desarrollo de software open source, la comunidad es la encargada del desarrollo y no solamente un desarrollador o una empresa. Uno de los casos más conocidos es el de Linux que es un sistema operativo open source, donde cualquier persona puede aportar al desarrollo del software del sistema o tambien descargar el código y crear sus propia versión. Los proyectos son publicados en GitHub donde se alojan la mayoria de proyectos open source.

En el campo de las bases de datos, el open source resulta ser el mismo concepto, un sistema de bases de datos cuyo código es abierto o público. Entre las bases de datos open source más populares se encuentran:

  • MySQL
  • PostgreSQL
  • MaxDB
  • Firebird
  • Ingres
  • MonetDB
  • LuciDb
